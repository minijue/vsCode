企业内部网Intranet(NT)文件系统技术

一、引言　　二、文件系统概述　　三、文件系统的模型　　四、安全问题

一、引言

信息时代的脚步声已临近。Internet正经历着快速的发展。越来越多的企业已接入Internet。事实上，企业更多的是需要在内部而不是在外部使用Web服务器，因此Intranet现在被形象地描述为"冰山"，正如你只能看到冰山的露出水面的一小部分一样，对于机构而言的外部Web服务器，实际上只是总的Web服务器的一小部分。Web服务器的主要部分像冰山的水中部分一样，隐含在防火墙内部，这些服务器用于内部信息的发布。

所谓Intranet，就是利用Internet技术把一个企业内的信息资源链接起来，这些资源包括文字、图片、信息数据库、工作流软件和文档管理等。企业使用企业内部网（Intranet）以帮助其员工查找信息、相互协作，并在整个组织内部分发合作成果。这里我们主要谈一谈企业内部网中文件生成、汇集及检索查询系统的建设。

二、文件系统概述

无论是什么样的企业单位，日常运作都要和大量的文件信息打交道，诸如信函、备忘录、各种工作计划、总结、市场需求信息、各种业务报表等等。尽可能科学有效地将各部门的各类文件组织起来，统一保存，以供全企业共享，不仅有助于领导及时了解、查询各部门的运作情况；有助于各部门规范管理自己的文件；有助于部门之间随时互通有无；更将积累成为企业的巨大信息财富，成为提高企业竞争力的重要因素。

通过对信息资源提供及时的访问，加速了企业内信息传播速度，使需要信息的人员在需要时能轻易得到信息，缩短了上市时间，进一步增加了产品质量和用户满意程度，同时减少了费用，最大程度地实现无纸办公。同时文件管理系统通过改善信息的可访问性、准确性、及时性和周转速度增强了企业竞争优势。概括起来可获得如下几项益处：

缩短上市时间；

改进产品质量；

提高企业生产力；

降低费用。

对大型企业而言，文件管理系统尤为重要，它是企业科学、规范管理的重要一步。通过它能够从企业内部共享文档集中，找到已经存在而且能重用的信息。

大多数文档经历下面四个基本阶段：

1.创建（Create）：各部门以不同的方式创建自己的文件，如文字处理软件、电子表格等；

2.浏览（Review）：在文件定稿之前，部门其他人员阅读、修改；

3.汇集（Assembly）：文件定稿后，上网存放在公共数据库里；

4.传播（Dissemination）：这时文件可供他人查询使用。

各部门将自己日常工作中所创建的各种文件，合理分类，审定后认为具有存档价值，可供其他部门共享，将其加入公共数据库，这样一个普通文件就成为共享文件信息系统中的一员了。


三、文件系统的模型

1.软件环境

服务器端软件系统我们采用：

Windows NT网络操作系统；

Microsoft Internet Information Server (IIS 3.0 or 4.0) 服务器软件；

JDK 1.1 Java语言；

Microsoft Access数据库软件。

客户端：

Windows 95操作系统；

IE或Netscape浏览器。

2.文件、数据库结构

用户建立的文件可以是文本或Word格式，当然也可以是HTML文档。使用记事本、Word 6.0或7.0等等。

采用Microsoft Access建立相关数据库，具体建库方法这里不再细述。

3.文件系统的实现

我们主要使用HTML超文本编辑语言与基于IIS的ASP (Active Server Pages)动态网站开发技巧，同时也运用了Java语言。

首先用HTML超文本编辑语言来编制大部分的浏览器界面，可以使用FrontPage编辑器，也可以用记事薄编写。

使用ASP主要用来实现与数据库的连接，动态查询、追加、删除数据库记录。ASP其实是一套微软开发的服务器端脚本环境，ASP内含于IIS 3.0和4.0之中，脚本语言用在定界符<％和％>之内，你可以在定界符之内使用任何有效的VB Script 命令，ASP会按VB Script处理这些命令。通过ASP我们可以结合HTML网页、ASP指令和ActiveX元件建立动态、交互且高效的WEB服务器应用程序。所有的程序都在服务器端执行，包括所有嵌在普通HTML中的脚本程序。当程序执行完毕后，服务器仅将执行的结果返回给客户浏览器，这样也就减轻了客户端浏览器的负担，大大提高了交互的速度。

Java用来实现从物理上删除服务器上的已无用文件。要编写服务器端应用程序和客户端Applet程序。服务器端应用程序在服务器端运行，随时监听客户端，收到请求后，即执行删除功能。客户端Applet程序发送客户端的请求，如请求删除的文件名。IIS 4.0版本中，利用ASP可完成上述功能。

4.文件系统功能

文件系统的基本功能包括：文件查询检索、文件追加与删除、标题检索、部门追加与删除、文件类型追加与删除。

文件查询检索：可按不同的部门、不同的文件类型检索各种文件，显示文件标题、文件名、日期和所属部门及文件类型。点击文件名后，查看文件的具体内容。

文件追加与删除：各部门依据各自的部门口令，有权限地将新文件目录加入网络数据库，并将原文件上传至服务器。同时也可删除服务器中已没有存在价值的旧文件。

标题检索：用户可以输入一个或几个关键字，用空格分割，系统帮助用户查找含有这些关键字的文件信息。

部门及文件类型追加与删除：这两个功能都只有系统管理员有权执行，当然是在部门或文件类型需要更改时才使用。

四、安全问题

在网络上运行，其数据安全性要求是必然的，特别是在网络联入Internet网的情况下。内外网间主要可以靠设置防火墙限制外部的访问。本文件系统内部主要根据用户口令来设置操作权限,通过验证用户输入口令的正误，来确定用户是否具有某操作的权利，用户口令的设置只有系统操作员才有权完成。当然，系统运行当中，更要时刻防范病毒的侵入，一旦网络中某台机器感染病毒，整个网络就有可能不能正常工作，所以，应当经常进行查毒工作。
