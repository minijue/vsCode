FrontPage98站点管理使用心得


“FrontPage”你一定听说过吧？自从被微软公司收购以后，它身价倍增，成为专业和非专业的网页设计人员必备的重要工具。虽然学习它比较简单，但是要充分发挥其强大功能并不容易。我们还是来听听高手的经验。也许你会发现原来“FrontPage”有这么多妙用，怎么一直没发现呢......
我曾经用过多种网页制作工具，如：从96年初类似于写字板的HTML Easy 1.0，到较早实现所见即所得的Navigator Gold 3.0 Editor，以及后来比较成熟的Hot Dog、Home Site等。但是自从有了FrontPage之后，我就变得非常专一了，因为它已经满足了我的多种需要，如：公司局域网上的站点、我的个人站点、公司的Internet站点、各种项目站点等等。并且我和同事可以安全、方便地更新它们而不必担心过去由于站点多、文件多而带来的各种问题。
FrontPage如此强大，我想不仅是因为它拥有一个成功的所见即所得的编辑器（Editor），或是集合了图形制作工具的Composer，根本原因还在于：它独有的完整的站点管理功能正是一个网页制作人长期工作中不可缺少的。
我想现在大多数主页制作人员安装在自己的PC里的一定是FrontPage98，因为这个产品在中文内码识别、模板、Form、Frame等功能上有了很大的提高。在这里，我们不妨构想一个这样的环境：一个20人的办公室，有20台PC终端的局域网，每台PC上都安装了FrontPage98。让我们来看看FrontPage98是如何帮助你实现站点管理的。
FrontPage98由FrontPage Server Administrator、FrontPage Explorer、FrontPage Editor三部分组成。其中Server 和Explorer的完美结合构成了它的站点管理的核心。这充分体现在它与站点相关的每一个环节上：建立站点、文件传输、浏览、站点维护。闲话少说，书归正传。那么现在还是让我们依次来体验FrontPage98的一些优点吧。

轻松的协作建站

FrontPage98允许你为项目建立一个Web，项目小组的人可以同时在这个Web上工作。
多数情况下，完成一个项目站点需要多人同时工作，如：站点策划人、图形设计师、制作人、资料提供人员等。我们怎样实现协调工作呢？一个有效的办法是用FrontPage98在其中一个人(往往是项目负责人)的PC上建立一个Web，然后从Server Administrator的Security或者Explorer的Tools-Permissions加入其他人对这个Web的User权限，这样每个小组成员可以同时在自己的PC上打开这个Web，并利用Explorer中的Task等工具（后面介绍）紧密配合完成工作。当然这一切的前提是这些PC是以某种方式相连了。

方便的文件传输

Publish的自动识别功能可以只传送最新更新的文件。我每天都要将本地的几个文件传送到远处的主机上完成一次网站的更新，在使用FrontPage之前，如果这些文件位于不同的4个文件夹内，那么我就要做四次传送的操作，这当然是一件重复而枯燥的工作。现在，有了FrontPage98后，就可以解脱了。在Explorer中你只要打开含有传送文件的那个Web，然后选择File下的Publish FrontPage Web功能，并在随后弹出的窗口中输入接收这些文件的地址（如图3），它就会自动将上次Publish后所有被更新或新建的文件传送到目的地。条件是你的远程主机上装有FrontPage Server Extension，并且你拥有目的地Root Web的用户权限。

快速浏览施工的网页

FrontPage Server 支持HTTP，它可以让你的PC成为一台Server。当你启动FrontPage Explorer时，FrontPage Personal Web Server会自动运行，这样你的PC就可以成为一台支持HTTP的Server。也就是说，在同一局域网内的任何一台PC上，别人都可以从浏览器上访问你的站点，地址是你的主机名加域名，例如:我的主机名是：sophie，域名是：newsoftware，那么访问地址就应该是：http://sophie.newsoftware/。这个地址将指向FrontPage安装的目录：c:\frontpage webs\content\index.htm，你可以从这里开始浏览你的个人站点。那么拨号用户的情况又是如何呢？从理论上讲，当你通过电话线连上Internet后，启动FrontPage Personal Web Server，你的站点就可以被来自Internet的朋友访问，只是你需要先告诉他们你拨号时获得的IP地址。

简单的站点维护

FrontPage98 Explorer提供了主要的站点维护功能,如：Web属性设置、链接正确性检测、Task任务设置等。你能否熟练使用这些功能是利用FrontPage提高工作效率的关键。

(1) Task
对于项目负责人来说，你可以利用这个功能给小组成员分配工作，告诉他们哪些文件需要修改或更新，并设置质量要求等级。具体操作如下：
a.设置Task
选中需分配的文件，点右键选“Add Task”，出现如图4的窗口，填写相关内容并点“OK”，即可将该文件设置到任务栏。
b.查看Task
打开Explorer的Task窗口就可以看到以红色圆点为标志的任务项。
c.完成任务
双击Task窗口中的某一个任务项，直接进入Editor编辑，存盘时会提示是否在任务栏将此文件设置为“已经完成”。

(2) 链接正确性检测
Hyperlinks和Hyperlink Status窗口以不同的方式告诉你检测结果。
Hyperlinks 显示每一个文件的链接关系，有间断的箭头线表示是一个断链。
Hyperlink Status 显示所有链接出错的文件。

(3) Web属性设置
打开“Tools”下“Web Settings”窗口（如图5）。这里可以设置当前 Web的名字、标题、以及默认的文字类型。

(4) Web权限设置
打开“Tools”下“Permissions”窗口（如图1）。这里你可以设置当前的Web是否使用与 相同的用户权限设置，如果不同，你也可以自行设置。用户权限有三个等级：Administer，Author和Browse。

服务器扩展

FrontPage Server Extensions 的功能是将相关的文件和目录安装到已经存在的Server软件中，让Server能够执行FrontPage的某些特性，例如从Form接受数据。FrontPage为多种流行的Server软件提供相应的Server Extentions，一部分包含在你购买的FrontPage软件包中，其他的可以从以下地址下载：http://www.microsoft.com/frontpage。
与一个安装有Frontpage Server Extensions的ISP的主机相连时， 你可以通过连接线路方便地编辑、校正和发布在你的Web上的文件。所有的功能都能正常运行，如：程序控制、图形Map等。
如果ISP的Server上没有FrontPage Server Extensions，你也可以在本地使用Microsoft Personal Web Server 来测试你的站点，如链接、图形和其他多媒体组件。但是，没有Extensions，如果你直接通过http打开和保存文件，就不能以 Client/Server 方式编辑，而且WebBot的功能也无法实现。

我使用FrontPage98已有一段时间了，个人认为FrontPage98是一个功能非常完整的站点制作及管理工具。它的售后服务也非常完善，除了在软件中提供有详尽的Help文件之外，微软公司还在它的站点开设了该产品的技术支持。最值得一提的是，它还有Newsgroup新闻组服务，你可以从中找到为FrontPage98的使用者们专门设立的讨论组，即使你没有购买它的正版产品也不会被拒之门外，该产品的几位开发人员仍然会热心地回答你的问题，并不定期地组织该产品的专题讨论。
